# 🗺️ User Journey (사용자 여정 맵)

사용자가 **Must Be The Apex**를 처음 설치하고 주행을 마칠 때까지의 흐름을 정의합니다.

### Step 1: 프로그램 구동 및 초기 설정
1. 유저가 제공된 시작 파일(`start.bat` 등)을 더블클릭하여 실행합니다.
2. 백그라운드에서 C# Agent와 Remix 서버가 동시에 구동됩니다.
3. 기본 웹 브라우저가 열리며 `http://localhost:3000` (대시보드)으로 자동 접속됩니다.
4. **(최초 1회)** 대시보드의 '설정' 화면에서 사용할 LLM 제공자(예: Claude)를 선택하고 API 키를 입력하여 저장합니다.

### Step 2: 주행 준비 및 모바일 연동
1. 브라우저 대시보드 화면에 "Waiting for F1 25 Telemetry..." 메시지가 표시됩니다.
2. 유저가 스마트폰 카메라로 화면 구석의 **QR 코드**를 스캔합니다.
3. 스마트폰 브라우저에 모바일 전용 미니 대시보드(기어, 속도 중심)가 나타납니다. 유저는 스마트폰을 휠 베이스에 거치합니다.

### Step 3: 주행 (실시간 텔레메트리)
1. F1 25 게임을 실행하고 Time Trial 또는 세션에 진입합니다.
2. 주행을 시작하자마자 PC 모니터의 미니맵에 차량 아이콘이 움직이며, 스마트폰에는 실시간 기어와 RPM이 반응성 있게 표출됩니다.
3. 코너를 지날 때마다 백그라운드에서 `lapDistance`를 바탕으로 주행 데이터가 태깅/기록됩니다.

### Step 4: AI 엔지니어 피드백 수신
1. 랩을 완료하고 결승선을 통과(Lap Completed)합니다.
2. Remix 서버가 즉시 방금 완료한 랩과 베스트 랩을 비교 연산하여 '손실 구간'을 추출하고 LLM에 API를 요청합니다.
3. 약 2~3초 후, PC 대시보드 중앙(혹은 게임 화면 밖 보조 모니터)에 AI 레이스 엔지니어의 피드백 팝업이 등장합니다.
   - *예: "Turn 4에서 브레이킹이 너무 이릅니다. 다음 랩에서는 50m 보드를 보고 브레이크를 밟아보세요."*
4. 피드백을 확인한 유저는 다음 랩 주행에 조언을 즉각 반영합니다.
