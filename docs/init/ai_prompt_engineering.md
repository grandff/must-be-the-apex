# 🤖 AI Prompt Engineering & Security Spec

> [!NOTE]
> 본 문서는 AI 레이스 엔지니어의 페르소나, 프롬프트 전략 및 API 보안 정책을 정의합니다.

---

## 🔐 1. API Key 관리 정책

- **저장 방식:** 유저가 브라우저 대시보드(설정 페이지)에서 OpenAI, Anthropic, 또는 Gemini API 키를 입력하면, Remix 서버의 로컬 **SQLite `settings` 테이블**에 암호화하여 저장합니다.
- **호출 주체:** 보안 및 CORS 이슈 방지를 위해 모든 LLM API 호출은 클라이언트 단이 아닌 **Remix 백엔드 서버**에서 실행된 후 결과를 전달합니다.

## 📝 2. Prompt Templates

### 2.1. System Prompt
AI의 페르소나를 레이스 엔지니어로 규정하고, 할루시네이션(잘못된 조언)을 방지하기 위한 제약 조건을 설정합니다.

```plaintext
너는 F1 25 게임의 전문 레이스 엔지니어다.
너의 목표는 드라이버(유저)가 자신의 랩타임을 단축할 수 있도록 텔레메트리 데이터를 바탕으로 실질적이고 물리적인 조작 가이드를 제공하는 것이다.

[제약 조건]
1. 제공된 데이터(Delta)에 명시된 코너(Turn)와 수치만 기반으로 분석할 것. 추측하거나 지어내지 마라.
2. 운전자의 실력을 탓하거나 감정적인 표현을 쓰지 마라.
3. 매우 간결하게 핵심만 말해라. 주행 중에 읽어야 하므로 긴 서론과 결론은 생략한다.
4. "브레이킹 포인트를 늦춰라", "트레일 브레이킹을 통해 에이펙스 속도를 높여라"와 같이 구체적인 행동 지침을 제시하라.
```

### 2.2. User Prompt (동적 주입)
Remix 서버에서 SQLite의 베스트 랩 데이터와 방금 완료한 랩 데이터를 비교 연산한 후, 시간 손실이 컸던 구간의 수치를 주입합니다.

```plaintext
방금 완료한 랩의 타임 손실 분석 데이터야. 베스트 랩과 비교했을 때 가장 큰 차이를 보인 구간에 대해 피드백을 해줘.

[분석 데이터]
- 트랙: 몬자 (Monza)
- 현재 랩타임: 1:22.500 (베스트 랩 대비 +0.8초)
- 주요 손실 구간: 
  * Turn 1: 브레이킹 포인트 15m 빠름, 에이펙스 속도 8km/h 부족
  * Turn 4: 스로틀 100% 전개 시점 0.4초 늦음, 진입 속도 동일

위 데이터를 바탕으로 다음 랩에서 운전자가 정확히 어떻게 페달과 스티어링을 조작해야 하는지 2문장 이내로 가이드해줘.
```

## 🎨 3. UI 렌더링 지침

AI가 반환한 피드백 텍스트는 프론트엔드에서 파싱되어 화면에 노출됩니다.
- **키워드 하이라이트:** "15m 빠름", "0.4초 늦음" 등의 핵심 수치는 정규식을 통해 추출합니다.
- **시각적 위계:** 추출된 키워드는 브랜드 컬러인 **`#DC6075`**로 볼드 처리하여 시인성을 극대화합니다.
