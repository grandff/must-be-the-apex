## 🏗️ 2. 시스템 아키텍처 및 기술 스택

데이터 수집과 화면 출력을 철저히 분리하여, 로컬 웹 환경 및 모바일 확장을 최우선으로 고려합니다.

### 📡 Data Ingestion (Windows Agent)
- **Role:** Windows PC 백그라운드에서 실행되는 경량 에이전트.
- **Function:** F1 25의 UDP 패킷을 수신하여 **WebSocket**으로 로컬 서버에 실시간 전송.

### 🌐 Frontend & Backend (Web Dashboard)
- **Framework:** `React Router + Remix`
- **Feature:** 에이전트 실행 시 `localhost` 브라우저 대시보드 자동 연결.

### 📱 Mobile Handoff (QR Sync)
- 대시보드 내 **로컬 IP 매핑 QR 코드** 제공.
- 동일 네트워크 내에서 별도 앱 설치 없이 스마트폰/태블릿으로 실시간 텔레메트리 화면 동기화.

## 🤖 4. 핵심 과제: AI 레이스 엔지니어 전략

방대한 시계열 데이터를 LLM에 직접 투입하는 대신, **'규칙 기반 연산 + LLM 자연어 생성'**의 하이브리드 구조를 채택하여 비용 최적화와 정확도를 확보합니다.

### 🛠 분석 파이프라인
1.  **Step 1. 로컬 데이터 전처리 (수치 연산)**
    - 레퍼런스 데이터(Best Lap)와 현재 유저 데이터를 비교.
    - *예: "Turn 4 진입 시 브레이킹 포인트 15m 빠름, Apex 속도 8km/h 부족"* 등 이상 지점 수치화.
2.  **Step 2. LLM 분석 리포트 생성**
    - 추출된 수치 요약본만 프롬프트로 구성하여 LLM에 전달.
    - 초심자가 이해하기 쉬운 **"현장감 있는 레이스 엔지니어의 언어"**로 가이드 생성.

> [!TIP]
> **특화 모델 도입에 대하여:** 초기에는 인프라 비용 절감을 위해 유저의 API 키를 활용한 LLM 연동 방식을 사용하고, 추후 데이터 축적 시 자체 경량 모델(SLM) 도입을 검토합니다.
